Ιανουάριος 2019
---------------
Ανατέθηκε στην εταιρεία "ΖΑΡΙΦΟΠΟΥΛΟΣ" να αναβαθμίσει το δίκτυο καρταναγνωστών
του ΝΔΜ και να το επεκτείνει στα κτίρια ΒΙΒ και ΚΙΘ. Η εταιρεία θα πρέπει, επίσης,
να καταργήσει τον παλαιό server ελέγχου που βρισκόταν στο control room του δευτέρου
ορόφου, και να τον επανεγκαταστήσει σε νέο (εικονικό) server του ΚΥΣ. Συμφωνήθηκε,
ακόμη, να εγκατασταθούν και τέσσερις νέες ηλεκτρονικές κλειδαριές σε επιλεγμένους
χώρους του ΝΔΜ. Η προθεσμία παράδοσης του έργου από την πλευρά της εταιρείας είναι
την Παρασκευή, 19 Απριλίου 2019.

Έγινε σύσκεψη με τον Ζώκα, τον Μελετιάδη, τον Στραβάκο και τον Παπαδόπουλο και
δρομολογήθηκαν οι απαραίτητες εργασίες.

Φεβρουάριος 2019
----------------
Δημιουργήθηκε ο control server (CS) στο ΚΥΣ από τον Στραβάκο και σε συνεργασία με τον
Ντούνια (Αθήνα) στήθηκε το πρόγραμμα της Honeywell.

Ο Νίκος Ιωαννίδης ασχολήθηκε με τον παλαιό CS στο control room του δευτέρου ορόφου.

Ο Λευτέρης Χριστοδούλου άλλαξε τους οκτώ (8) controllers και έβαλε τις ηλεκτρονικές
κλειδαριές εκεί που έλειπαν.

Ο Χριστοδούλου με τον Κλετσίδη μετέβησαν στο ΒΙΒ και το ΚΙΣ προκειμένου να επιλεγούν
τα σημεία τοποθέτησης των εκεί controllers και καρταναγνωστών.

Ο Χριστοδούλου τοποθέτησε τους controllers και τους καρταναγνώστες στα κτίρια ΒΙΒ και
ΚΙΣ.

Επειδή οι κάρτες εισόδου/εξόδου χρησιμοποιούνται και ως κάρτες πρόσβασης, εκδόθηκε και
κυκλοφόρησε εγκύκλιος υπ' αρ. 507/18066/18-2-2019 από τον Γενικό Γραμματέα με την οποία,
καλούνται οι Υπηρεσίες να αποστείλουν στη Διεύθυνση Προσωπικού τις υπάρχουσες κάρτες
πρόσβασης, διότι όλες οι προσβάσεις θα καταργηθούν και θα δοθούν εκ νέου.

Η Καργάκη και ο Παπαδόπουλος δημιούργησαν repository για τα προγράμματα εφαρμογών:

https://panos1962@bitbucket.org/panos1962/kartel

Τρίτη, 6 Αυγούστου 2019
-----------------------
Σε συνέχεια όχλησης που είχα από τον Νίκο Μπακάλη σχετικά με «χτυπήματα» για τα
οποία εστάλησαν ενημερωτικά μηνύματα αλλά δεν είχαν πραγματοποιηθεί, ήλεγξα την
κάρτα της Φανής Γιαννούση (50004) η οποία ήταν μια από τις κάρτες που παρουσίαζαν
αυτό το φαινόμενο. Μετά από ενδελεχή έρευνα με τη βοήθεια του Θέμη και της Δώρας,
καταλήξαμε στον επιπλέον περιορισμό των [History] rows με βάση το πεδίο [Type1]
το οποίο θα πρέπει να είναι 9, και το [Type2] που θέλουμε να είναι 0. Διορθώθηκε
το σχετικό SQL script του προγράμματος kartel.

Δευτέρα, 15 Απριλίου 2024
-------------------------
Άλλαξε η διαδικασία ενημέρωσης δεδομένων προσωπικού από το ΟΠΣΟΥ. Πράγματι,
μετά την εφαρμογή του νέου ΟΠΣΟΥ (18 Ιανουαρίου 2024), η διαδικασία ενημέρωσης
δεδομένων προσωπικού έπαψε να λειτουργεί. Η διαδικασία είχε ως εξής: Κάθε μέρα,
και ώρα 03:11, έτρεχε cronjob το οποίο εκτελούσε SQL scripts στον database
server του ΟΠΣΟΥ (ORACLE) και τα αποτελέσματα φορτώνονταν στις database
"erpota1" και "erpota2" εναλλάξ.

Από τις 18 Ιανουαρίου 2024 δεν υπάρχει πια η δυνατότητα να «τρέξουμε» SQL
queries από τους δικούς servers. Η λύση που επιλέχθηκε είναι να τρέχουν τα
SQL queries στον database server του νέου ΟΠΣΟΥ (prosgtreSQL) και να
μεταφέρονται τα αποτελέσματα στον kartel server μέσω scp. Για τον σκοπό αυτό,
δημιουργήθηκε χρήστης "np" στον kartel server και δόθηκαν τα cerdentials στην
εταιρεία Neuropublic, προκειμένου να μπορεί να γίνεται η αποστολή των αρχείων
μέσω scp.
