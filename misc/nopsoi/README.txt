Στο παρόν directory υπάρχουν προγράμματα και άλλα αρχεία που εξυπηρετούν στην
ενημέρωση των στοιχείων προσωπικού από το ΟΠΣΟΥ. Ουσιαστικά, πρόκειται για τις
database "erpota1" και "erpota2" που ενημερώνονται εναλλάξ καθημερινά, δηλαδή
τη μια μέρα ενημερώνεται η database "erpota1" και την άλλη μέρα ενημερώνεται
η database "erpota2". Παράλληλα ενημερώνεται η σχετική παράμετρος στην database
"kartel" με τον αριθμό 1 ή 2, ανάλογα με το ποια είναι η πιο «φρέσκια» database
από τις "erpota1" και "erpota2".

Στο παλαιό ΟΠΣΟΥ το πρόγραμμα της ενημέρωσης «έτρεχε» κάθε βράδυ μέσω cron με
τα credentials του χρήστη "kartel". Η διαδικασία είχε τα εξής βήματα:

	Εκτέλεση SQL queries στον database server του παλαιού ΟΠΣΟΥ (ORACLE).

	Λήψη των αποτελεσμάτων των queries στα αρχεία "ipalilos.tsv" και
	"metavoli.tsv" στον kartel server.

	Ενημέρωση της database "erpota1" ή "erpota2" από τα παραπάνω αρχεία.

	Αλλαγή της παραμέτρου "kartel" που δείχνει ποια από τις databases
	"erpota1" και "erpota2" είναι η τρέχουσα.

Στο νέο ΟΠΣΟΥ δεν υπάρχει πια η δυνατότητα εκτέλεσης SQL queries από τον kartel
server. Έτσι, επιλεθηκε η λύση να τρέχουν τα SQL queries στον database server
του νέου ΟΠΣΟΥ (ProstgreSQL) και να αποστέλλονται τα αποτελέσματα στα αρχεία
"ipalilos.txt" και "metavoli.txt" στον kartel server μέσω scp. Για τη μεταφορά
των αρχείων δημιουργήθηκε χρήστης "np" στον kartel server και δόθηκαν τα
credentials στην εταιρεία Neuropublic.

Τα αρχεία μεταφέρονται κάθε βράδυ μέσω cron, από τον database server του ΟΠΣΟΥ
στο home directory του χρήστη "np". Σε εύλογο χρονικό διάστημα τα αρχεία
ελέγχονται και μετατρέπονται σε μορφή βολική για να φορτωθούν στις databases
"erpota1" και "erpota2". Λίγο αργότερα εκτελείται και πάλι η παλαιά διαδικασία,
όπου το πρώτο βήμα έχει αντικατασταθεί με λήψη των αρχείων "ipalilos.tsv" και
"metavoli.tsv" απευθείας από το home directory του χρήστη "np".

Τα εκτελέσιμα αρχεία "erpotapp.sh" και "txt2tsv.sh" γίνονται link στο bin
directory του χρήστη "np", ενώ το αρχείο "crontab" πρέπει να φορτωθεί στο
cron του χρήστη "np".
